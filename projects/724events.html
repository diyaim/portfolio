724events

Campagne de tests QA : cahier de recette, tests exploratoires, tickets Jira et analyse API via Swagger.

Retour aux projets Documentation

Contexte

Je suis QA testeur dans une agence web en charge du site de l’agence événementielle 724events. Une première version du site a été livrée et il est temps de lancer une campagne de tests.

Le projet s’appuie sur un conteneur Docker (site en local), un cahier de recette sur Notion, des tests exploratoires, et l’analyse de l’API via Swagger.

Objectifs

  • Exécuter le cahier de recette et renseigner le statut de chaque test.
  • Compléter avec des tests exploratoires (données, UI, navigateurs…).
  • Analyser les anomalies (console, logs, Swagger si besoin).
  • Créer des tickets sur Jira (criticité + preuves) et exporter le board en CSV.
  • Rédiger un bilan de campagne (GO / NO GO + synthèse).

Réalisation

  • Mise en place de l’environnement de test (Docker + accès au site en local).
  • Création d’une instance Jira et utilisation du board par défaut pour les bugs.
  • Accès à Swagger et vérification des endpoints (ex : GET /events/).
  • Exécution du cahier de recette (Notion) + analyse immédiate des anomalies détectées.
  • Tests exploratoires chronométrés (données : 20 min, tests libres : 30 min).
  • Création des tickets Jira : description, étapes, attendu/obtenu, criticité, captures.
  • Mise à jour du cahier de recette pour les scénarios exploratoires + lien ticket Jira.
  • Rédaction du bilan de campagne : objectifs, résultats, anomalies, décision GO/NO GO.
